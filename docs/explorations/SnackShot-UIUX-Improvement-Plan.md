# SnackShot UI/UX 개선 계획안

> 등급: 탐색/제안(exploration). 권위 문서가 아니며, 채택 항목은 ADR·DesignSystem 갱신을 거쳐 확정한다(`docs/INDEX.md`).
> 작성일: 2026-06-19
> 범위: 사용자 요청에 따라 **① 캡처/녹화 플로우 · ② 기록 열람/검색(아카이브) · ③ 디자인 시스템 일관성** 3개 축에 집중. AI 의사결정 보조는 향후 과제로 별도 정리.
> 근거: 현 코드(`app/`, `src/theme/`, `src/components/`) 정독 + 유사 서비스 조사(문서 말미 출처).

## 0. 요약 (TL;DR)

SnackShot은 "종이 다이어리" 메타포와 토큰 기반 디자인 시스템이 이미 탄탄하다(앱 코드 내 하드코딩 색상 0건, 폰트 로드 완료). 약점은 **시각**보다 **흐름(flow)**에 있다.

- **캡처**: 핵심 가치인 "간편한 녹음/녹화"에 비해 진입 경로가 Today 탭의 하단 툴바 한 곳뿐이고, 저장 전 결정(모드·힌트·메모)이 많아 마찰이 있다.
- **아카이브**: 강력한 FTS 검색을 갖췄지만 필터가 없고, "날짜를 먼저 골라야" 과거를 본다 — 연속 회고(타임라인) 경로가 비어 있다. 모드 전환 제스처가 숨겨져 있다.
- **디자인 시스템**: 토큰 규율은 모범적이나, 문서-코드 드리프트(폰트 로드맵 등), 다크/야간 모드 부재, 햅틱·모션 적용 일관성 미검증이 남는다.

우선순위: **P0 — 글로벌 퀵캡처 + 저장 마찰 축소 + 아카이브 필터**, P1 — 회고 타임라인·검색 강화·온보딩, P2 — 다크모드·햅틱·모션 정합·AI 보조 진입점.

## 1. 유사 서비스 조사

### 1.1 카테고리별 포지셔닝

| 서비스 | 분류 | 핵심 UX 강점 | SnackShot이 취할 점 |
|--------|------|--------------|---------------------|
| **Day One** (Gold, 2026) | 텍스트-우선 멀티미디어 저널 | 방해 없는 빈 페이지 에디터, **Today 탭**(하루 단위 허브: On This Day·Daily Chat·Moments), AI 요약/프롬프트 | "오늘" 단일 허브 + 과거 회상(On This Day) 패턴 |
| **1 Second Everyday** | 영상 일기 | **캘린더 메인 UI**(채운 날/빈 날 시각화), 하루 여러 스니펫, 창의적 리마인더, 영상→무비 합성 | 캘린더 채움/빈칸 대비, 리마인더로 습관화 |
| **Speakwise / Lound / MindDrop** | 음성-우선 저널 | **원탭 녹음**, AirPods 핸즈프리, 온디바이스 STT(프라이버시), 수개월 패턴 분석 | 마찰 없는 원탭 캡처, 온디바이스/프라이버시 메시징 |
| **Decision Journal** (Farnam St 계열) | 의사결정 저널 | 결정+근거+**확신도(저~고)** 기록, **예약 알림으로 결과 회수**(예: 7일 후), 한 화면 로그 리뷰 | 확신도·후속 알림·결과 회수 루프(이미 부분 구현) |

### 1.2 2026 음성/영상 저널 UX의 공통 합의

- **"음성을 받는 앱" vs "음성을 위한 앱"의 구분.** 후자는 UI·AI·기능 전부가 "말하기"를 1차 입력으로 가정한다. SnackShot은 후자를 지향하지만, 진입은 아직 텍스트 메모 화면(Today)에 종속돼 있다.
- **첫 산출물까지의 속도가 곧 'Aha'.** Loom/Voice Memos류는 계정·설정·튜토리얼 없이 곧장 녹음으로 들어간다. "하는 것"에서 보상이 나오는 도구일수록 온보딩은 읽기가 아니라 즉시 실행이어야 한다.
- **위젯/원탭 진입**이 표준. 아이디어가 떠오른 순간의 마찰이 곧 이탈이다.
- **결과 회수 루프**(결정→일정 후 알림→결과 기록)가 의사결정 저널의 차별화 핵심. SnackShot의 Inbox 보드(후속 확인/회고)가 이미 이 방향이라 강점이다.

## 2. 현재 SnackShot 분석

### 2.1 구조 스냅샷

- **탭 4개**: Today(하루 단위 일기·작성) · Archive(캘린더+FTS 검색) · Inbox(AI 추출 컨펌 덱 + 결정 보드) · Settings.
- **캡처 경로**: Today 하단 `TodayComposer`(업로드/음성/영상) → `record`/`record-audio` → `preview`(모드·힌트·메모) → 저장 → Today. 텍스트는 Today 인라인 입력.
- **파이프라인**: 저장 후 `ai_jobs` 큐(압축→STT→라벨추출→옵시디언 export)가 비동기로 돌고, Today/Inbox가 폴링·반영.

### 2.2 강점

1. **디자인 토큰 규율이 모범적.** `app/`·`src/components/` 전체 하드코딩 hex **0건**(grep 확인). 색·간격·라운드·그림자·타이포가 전부 semantic 토큰 경유. 디자인 변경 비용이 매우 낮은 상태.
2. **일관된 메타포.** 종이·폴라로이드·압정·테이프·손글씨(Gaegu)+산세리프(NotoSansKR)로 정서적 일관성. 폰트도 이미 로드 완료(`app/_layout.tsx`).
3. **색각 보조 설계.** 확신도를 막대+수치, 카테고리를 색+텍스트로 이중 표기 — 접근성 기본기가 좋다.
4. **캡처 안전장치.** 3초 미만 미저장·3분 상한·녹화 실패 graceful 안내·저장 중 취소 잠금 등 엣지케이스 처리가 견고.
5. **결과 회수 루프 선점.** Inbox의 후속 확인/회고 보드는 경쟁 의사결정 저널의 핵심 패턴을 이미 구현.

### 2.3 약점 — 축별 진단

#### A. 캡처/녹화 플로우

- **진입점이 Today 한 곳.** 다른 탭(Archive/Inbox/Settings)에서 떠오른 순간을 바로 못 잡는다. "간편 캡처"라는 가치와 충돌. (경쟁 합의: 원탭/위젯 진입)
- **저장 전 결정 과다.** `preview`에서 모드(독백/조용)·중요결정 체크·메모를 모두 본 뒤 저장. 빠른 한 컷에는 마찰이다. 게다가 핵심 가치인 **"중요 결정 포함" 체크박스가 메모와 같은 위계로 묻혀** 발견성이 낮다 — 결정 추출이 켜지는지 사용자가 의식하기 어렵다.
- **모드 선택 시점이 사후.** 녹화가 끝난 뒤 "독백/조용"을 고른다. 라벨도 추상적("조용 모드"=STT 미실행인지 불명확). 녹화 전 의도가 정해지는 경우가 많다.
- **음성 녹음 중 피드백 부재.** 파형/레벨 미터가 없으면 "녹음되고 있나?"라는 불안이 생긴다(음성-우선 앱의 기본 요소).
- **일시정지/이어찍기 없음.** 한 호흡으로만 녹화 가능.

#### B. 기록 열람/검색 (아카이브)

- **연속 회고(타임라인) 경로 부재.** 과거를 보려면 **반드시 날짜를 먼저 선택**해야 한다. "그냥 쭉 거슬러 보기"(역시간순 피드)나 Day One의 "On This Day(작년 오늘)" 같은 회상 진입이 없다.
- **검색 필터 0개.** FTS는 강력하나 타입(영상/음성/텍스트)·결정 포함 여부·카테고리·기간으로 좁힐 수 없다. 텍스트 일치만 가능.
- **모드 전환이 숨은 제스처.** 캘린더 month→compact→week 순환이 "날짜 헤더 탭"이라는 비가시 인터랙션. chevron 아이콘만으로는 발견성이 낮다.
- **빈 상태가 밋밋함.** 모두 단색 텍스트 한 줄. 첫 사용자에게 다음 행동을 안내하지 못한다(캡처 CTA 부재).
- **결정만 모아 보기**가 별도 `decisions` 화면으로 분리돼, 아카이브에서 "결정의 흐름"을 시각적으로 못 훑는다.

#### C. 디자인 시스템 일관성

- **문서-코드 드리프트.** `SnackShot-DesignSystem.md` §10 적용 로드맵 1번이 "폰트 로드(미완)"로 적혀 있으나 실제론 완료. 권위 문서가 현실과 어긋나면 신뢰가 깎인다(프로젝트가 가장 경계하는 drift).
- **단일 라이트 테마.** 야간 녹화·OLED·취침 전 기록 맥락에서 다크 모드 수요가 크나 부재. media(어두운) 토큰은 이미 있어 확장 토대는 있음.
- **모션·햅틱 적용 미검증.** 모션 토큰(duration/easing)은 정의됐지만 화면 전환·시트·스와이프에 일관 적용됐는지 불명. 녹화 시작/정지·컨펌/기각 스와이프에 **햅틱 피드백 부재**(촉각 보상은 캡처 도구의 기본 만족 요소).
- **아이콘 톤 불일치 가능성.** Ionicons(범용 라인 아이콘)가 종이·손글씨 정서와 미세하게 어긋난다. 강조 영역(탭바·주요 액션)부터 톤 정합 검토 여지.

## 3. 개선안

### 3.1 캡처/녹화 플로우

1. **글로벌 퀵캡처 진입(P0).** 어느 탭에서도 닿는 단일 캡처 진입점을 둔다. 선택지:
   - (a) 탭바 중앙의 캡처 FAB/중앙 버튼(녹화·음성·텍스트를 long-press 또는 미니 시트로 분기),
   - (b) 또는 홈 위젯/잠금화면 단축어(Expo 한계 확인 필요).
   - 권장: 우선 (a)로 앱 내 1탭 도달을 확보하고, 위젯은 P2.
2. **저장 마찰 축소(P0).** `preview`에 **"바로 저장"** 기본 경로를 둔다. 모드는 합리적 기본값(독백/voice)으로 선저장, 메모·힌트는 "나중에/세부 펼치기"로 접는다. 저장 후 Today 카드에서 인라인 보강 가능하게.
3. **"중요 결정" 발견성 격상(P1).** 체크박스를 메모와 분리해, 압정/하이라이트가 붙은 **별도 토글 블록**으로 승격(DesignSystem의 "결정 발견 배너" 스타일 재사용). 켜짐 상태를 색+아이콘+문구로 명시.
4. **모드 의도를 캡처 진입에 통합(P1).** 음성/영상 진입 시 "독백(말하며 기록)/조용(영상만)"을 **시작 화면에서** 1탭으로 정하거나, 라벨을 행위 중심으로 재서술("말로 남기기"/"소리 없이"). STT 실행 여부와의 관계를 마이크로카피로 1줄 설명.
5. **음성 녹음 파형·레벨 미터(P1).** `record-audio`에 실시간 진폭 시각화 추가 — "녹음 중" 신뢰 확보.
6. **녹화 일시정지/이어찍기(P2).** 한 모먼트를 여러 호흡으로. ADR-005(3분 상한)와 합산 규칙만 정의하면 됨.

### 3.2 기록 열람/검색 (아카이브)

1. **회고 타임라인 뷰 추가(P0).** 날짜 선택과 **병렬**로 "역시간순 연속 피드"를 둔다(아카이브 상단 세그먼트: `캘린더 | 타임라인`). 끝없이 거슬러 스크롤하며 폴라로이드 카드를 훑는 1SE/Day One식 경험.
2. **검색 필터 칩(P0).** 검색바 하단에 타입(영상/음성/텍스트)·결정 포함·기간 칩. FTS 쿼리에 `WHERE` 조건만 추가 — DB 비용 낮음(ADR-010/013 호환).
3. **"On This Day(작년 오늘)" 회상(P1).** Today 또는 아카이브 상단에 "1년 전 오늘"·"한 달 전" 카드. 습관화·정서 가치가 큰 경쟁 표준 기능.
4. **모드 전환 가시화(P1).** 숨은 헤더 탭 대신 명시적 세그먼트 컨트롤(월/주)로 교체. 압축 모드는 자동(스크롤)으로.
5. **빈 상태에 CTA(P1).** "이번 달엔 아직 기록이 없어요" → 곧장 캡처로 가는 버튼·일러스트(종이 메타포). 첫 사용자 이탈 방지.
6. **결정 흐름 인레이(P2).** 아카이브 타임라인에 결정 카드를 같은 축에 섞어, 일기와 결정을 하나의 시간축에서 회고.

### 3.3 디자인 시스템 일관성

1. **문서-코드 정합 1회 정리(P0, 저비용).** DesignSystem §5/§10의 "폰트 미완" 서술을 "완료(Gaegu/NotoSansKR via @expo-google-fonts)"로 현행화. drift 차단은 이 프로젝트의 최우선 가치.
2. **다크/야간 테마(P1).** semantic 토큰을 light/dark 2세트로 확장(원시 palette는 유지, 매핑만 분기). media 토큰이 이미 어두워 카메라·프리뷰는 무변경. 시스템 설정 연동 + 수동 토글.
3. **햅틱 토큰화(P1).** `expo-haptics`를 얇게 감싼 `feedback` 유틸 도입 → 녹화 시작/정지, 컨펌/기각 스와이프, 저장 완료에 일관 적용. 디자인 시스템에 "촉각" 절 추가.
4. **모션 적용 감사(P2).** 정의된 duration/easing 토큰이 시트·스와이프·화면 전환에 실제로 쓰이는지 점검하고 누락 보강. 단일 진입 헬퍼로 통일.
5. **아이콘 톤 정합 검토(P2).** 탭바·주요 액션부터 종이 정서에 맞는 아이콘 세트(또는 굵기/라운드 보정)를 시범 적용해 A/B.

### 3.4 (향후) AI 의사결정 보조 진입점

> 본 계획의 핵심 3축은 아니지만, 다가올 기능을 UI가 미리 수용하도록 자리만 잡아둔다.

- 결정 작성/컨펌 시점에 **"비슷한 과거 결정 N건"**을 같은 화면 하단에 인레이로 노출(별도 탭 신설 자제 — 맥락 안에서 보조).
- Inbox 결정 카드에 "지난 결정과 비교" 액션. 확신도·결과(성공/실패) 히스토리를 근거로 제시.
- 신뢰성 메시징: 추천이 아니라 **"참고용 과거 패턴"**임을 마이크로카피로 명확히(과신 방지).

## 4. 우선순위 로드맵

| 순위 | 항목 | 축 | 기대효과 | 대략 난이도 |
|------|------|----|----------|-------------|
| **P0** | 글로벌 퀵캡처 진입(탭바 캡처 버튼) | 캡처 | 핵심 가치 "간편 캡처" 실현, 이탈↓ | 중 |
| **P0** | preview "바로 저장" 기본화 | 캡처 | 첫 산출물까지 속도↑ | 소 |
| **P0** | 아카이브 회고 타임라인 뷰 | 아카이브 | 과거 열람 경로 신설 | 중 |
| **P0** | 검색 필터 칩(타입/결정/기간) | 아카이브 | 검색 정밀도↑ | 소~중 |
| **P0** | DesignSystem 문서 현행화 | 디자인 | drift 차단(즉시) | 소 |
| **P1** | "중요 결정" 토글 격상 | 캡처 | 결정 추출 발견성↑ | 소 |
| **P1** | 음성 파형·레벨 미터 | 캡처 | 녹음 신뢰 확보 | 중 |
| **P1** | On This Day 회상 카드 | 아카이브 | 습관화·정서 가치 | 소~중 |
| **P1** | 빈 상태 CTA·모드 세그먼트 | 아카이브 | 첫 사용자 안내 | 소 |
| **P1** | 다크/야간 테마 | 디자인 | 야간 사용성·접근성 | 중 |
| **P1** | 햅틱 토큰·적용 | 디자인 | 촉각 만족감 | 소~중 |
| **P2** | 녹화 일시정지 / 모션 감사 / 아이콘 톤 / AI 보조 인레이 | 전축 | 완성도·차별화 | 중~대 |

### 권장 진행 방식

1. **P0의 "문서 현행화"와 "바로 저장"부터** — 저비용·고확실. 즉시 체감.
2. **퀵캡처 + 타임라인 + 검색 필터**를 한 스프린트로 묶어 "캡처→열람" 순환을 완성.
3. 각 항목은 `feature-dev` 하네스로 ADR 준수 구현, `npm run verify` 게이트 통과 후 에뮬레이터 시각 확인.
4. 디자인 토큰 신규값(다크 매핑·햅틱)은 **토큰에 먼저 추가 후 참조**(하드코딩 금지 규칙 유지).

> 채택 시 ADR(예: 퀵캡처 진입 = 신규 ADR, 다크 테마 = DesignSystem 개정)을 갱신하고 `docs/INDEX.md` 권위표에 반영한다.
