## 0. 요약 (TL;DR)

SnackShot은 시각 토대가 이미 우수하다. 토큰 규율(하드코딩 hex 0건), 종이 메타포, 손글씨(Gaegu)+산세리프(NotoSansKR) 위계, 색각 보조 설계가 모두 자리 잡혀 있다. 6/19 계획의 흐름 개선 P0(글로벌 퀵캡처 FAB·아카이브 타임라인 세그먼트·캘린더 썸네일 등)도 **상당수 이미 구현돼 에뮬레이터에서 확인**됐다.

따라서 지금 가장 큰 ROI는 **"잘 만든 종이 시스템"을 "손때 묻은 다이어리"로 끌어올리는 마감(craft)과 운동감(motion)**에 있다. 정의만 돼 있고 화면엔 덜 산 자산이 많다:

- **감성 자산 미활용.** 테이프·압정·형광펜·기울임·손글씨가 토큰엔 있으나, 실제 화면은 폴라로이드 기울임·캘린더 썸네일 정도로 얕다. 헤더·빈 상태·카드가 "깨끗한 앱"에 머문다.
- **모션 거의 없음.** `react-native-reanimated@4`·`gesture-handler`가 설치돼 있지만 **실사용은 `DecisionDeck` 단 1곳.** 전환·등장·확정 피드백이 정적이다. 모션 토큰(duration/easing)도 대부분 미사용.
- **햅틱 배선만 됨.** `@/lib/haptics`에 의미 어휘가 다 깔려 있으나 `expo-haptics` 미설치로 전부 no-op. 촉각 보상이 0.
- **정보구조 디테일.** 헤더 과대(34px)·Today 날짜 중복·이동 버튼 시인성 등 6/19에서 지적된 저비용 항목 일부가 아직 화면에 남아 있다(에뮬 확인).

핵심 메시지: **새 기능보다 "마감 한 패스"가 정서적 차별화를 만든다.** 아래 3축은 대부분 토큰·공용 컴포넌트 한 곳 수정으로 전 화면에 퍼지는 저비용·고체감 작업이다.

## 1. 유사 서비스 · 트렌드 리서치 (감성/모션 관점)

6/19 문서가 경쟁 앱의 **흐름**(Day One·1SE·음성저널·Decision Journal)을 정리했으므로, 여기서는 **질감·모션** 축만 보강한다.

### 1.1 2026 비주얼 트렌드 — "촉각적 불완전함"이 진정성

2026 디자인 담론의 합의는 **scrapbook/collage 에너지의 귀환**이다. 종이 질감, 손그림 요소, 콜라주식 겹침, 의도적 노이즈·불규칙이 "진짜 사람이 만든 것"의 신호로 쓰인다. 순수 스큐어모피즘은 낡았지만, **flat 토대 위에 깊이·질감 단서를 선택적으로 얹는 하이브리드**가 가장 성공적이다. SnackShot의 "종이 위 폴라로이드/스티키" 메타포는 정확히 이 하이브리드 지점에 있다 — 더 밀어붙일 여지가 충분하다.

함의: 균일한 그리드·완벽한 정렬을 일부러 깨고(±1~2° 기울임, 손그림 구분선, 종이 결), 장식을 "기록을 꽂아둔 흔적"으로 의미화하면 경쟁 앱과 정서적으로 갈린다.

### 1.2 2026 모션 트렌드 — 물리 기반 + 맥락 햅틱

모바일 모션의 표준이 "장식적 트랜지션"에서 **물리 기반(스프링)·맥락 햅틱·의미 있는 운동**으로 이동했다. 타이밍 합의: **탭 등 마이크로 100–250ms, 콘텐츠 전환 300–600ms.** 새로고침이 고무줄처럼 늘어났다 튕겨 체크로 변하고 미세 진동이 동반되는 식의 "압축 제스처 + 피드백 레이어"가 기대치가 됐다. Asana의 완료 유니콘처럼 **완수 순간의 작은 보상**이 반복 사용 동기를 만든다.

함의: SnackShot의 핵심 순간 — **저장 완료·결정 컨펌·녹화 시작/정지** — 에 스프링 등장 + 햅틱 + 작은 축하를 얹으면, "도구를 쓰는 손맛"이 생긴다. 토대(Reanimated·haptics 어휘)는 이미 깔려 있어 비용이 낮다.

## 2. 현재 상태 진단 (코드 + 에뮬레이터 실측)

### 2.1 에뮬레이터에서 본 4탭

- **Today**: 상단 캡션 `2026년 6월 20일 토요일` + 큰 손글씨 `‹ 오늘 ›` + `목록 보기`·검색. 빈 상태 "오늘의 첫 스냅샷을 남겨보세요"(단색 1줄). 하단 "오늘 한 줄, 직접 쓰기…" + 캡처바(업로드/음성/영상, 영상=테라코타).
- **Archive**: `Archive`(산세리프 대제목) + `이번 달 32개`. 검색바 + 세그먼트 칩(캘린더/주/타임라인). 캘린더는 **기록 있는 날에 미니 썸네일+개수 배지**가 이미 들어가 있다(점이 아님 — 6/19 제안 일부 반영됨). 오늘(20) 테라 원형 강조.
- **Inbox**: `Inbox` + `진행 중 1 · 회고 2 · 후속 0` + 세그먼트(현재 todo/전체 목록). 결정 카드: 스티키 태그(커리어)·예정일·제목·본문·결과 버튼(좋음👍/아쉬움😞). 완료 2건은 취소선 행.
- **Settings**: `설정` + 접이식 섹션(용량 관리/옵시디언 연동/영상 백업/자동 영상 관리/API 키). 정갈한 리스트.
- **전역**: 비Today 탭에 글로벌 퀵캡처 FAB(테라 +)가 실제로 떠 있음 — `QuickCaptureFab.tsx`로 구현 완료.

### 2.2 강점 (유지·강화할 자산)

1. **토큰 규율.** 색·간격·라운드·그림자·타이포가 전부 semantic 토큰 경유. **감성/모션 변경이 전 화면에 저비용으로 전파**되는 구조 — 본 계획의 실행 가능성이 높은 근본 이유.
2. **메타포 일관성.** 종이·폴라로이드·압정·테이프·손글씨가 토큰·컴포넌트(`Pin`·`Tape`·`Polaroid`)로 이미 존재. "정의는 끝, 적용만 남음" 상태.
3. **모션/햅틱 토대 설치 완료.** Reanimated 4·gesture-handler·react-native-svg 설치됨. `@/lib/motion`(layoutAnimate)·`@/lib/haptics`(의미 어휘) 단일 진입점도 배선됨. **불을 켜기만 하면 되는** 상태.
4. **색각 보조.** 확신도=막대+수치, 카테고리=색+텍스트 이중 표기.

### 2.3 약점 — 3축 진단

#### A. 핸드메이드 감성 (정의됐으나 화면에 덜 삶)

- **헤더가 "앱"답다.** Archive/Inbox/설정 대제목이 산세리프 볼드 대형. 손글씨(Gaegu)·형광펜·종이 장식이 헤더에서 빠져 "노트"보다 "대시보드"로 읽힌다.
- **빈 상태가 밋밋.** Today 빈 화면이 단색 텍스트 1줄. 종이 메타포(손그림 일러스트·점선 자리표시·압정)와 CTA가 없어 첫인상이 차갑다.
- **카드 장식 얕음.** `Pin`/`Tape` 컴포넌트가 있는데 모먼트·결정 카드에 거의 안 붙는다. 폴라로이드도 정적 기울임만, "꽂아둔" 입체감(겹침·그림자 변주)이 약하다.
- **종이 질감 단조.** 캔버스가 단색 모래색. 도트/섬유 결, 가장자리 음영, 종이 사이 그림자 같은 "표면감"이 없어 평평하다.
- **구분선·아이콘이 기계적.** Ionicons 라인·직선 hairline이 손글씨 정서와 미세하게 어긋난다.

#### B. 애니메이션/모션 (거의 전무)

- **실사용 1곳.** `Animated`/Reanimated 사용처가 `DecisionDeck` 뿐. 탭 전환·화면 진입·리스트 등장·시트 오픈이 즉시 점프(또는 OS 기본)라 정적이다.
- **확정 순간 무보상.** 저장·컨펌·기각·녹화 토글에 시각 축하도 햅틱도 없음(haptics no-op). 핵심 도파민 지점이 비어 있다.
- **상태 전이 끊김.** 빈→로딩→채움, 잡 큐 진행(압축/STT/라벨) 같은 비동기 상태가 페이드/스켈레톤 없이 튄다.
- **제스처 빈약.** 좌우 스와이프(Inbox 덱) 외엔 당겨서 새로고침·스와이프 액션·롱프레스 미리보기 등이 없다.

#### C. 정보구조/사용흐름 (저비용 디테일 잔존)

- **헤더 타이포 과대.** `displayLarge` 34px 일괄(6/19 지적). 손글씨는 대형일수록 가독성↓.
- **Today 날짜 중복.** 캡션 `…6월 20일 토요일` + 디스플레이 `오늘`(+과거일 땐 날짜 재표기) → 월·일 중복 가능.
- **이동/네비 버튼 시인성.** Today chevron이 연회색 소형, Archive 월이동 화살표가 작아 44pt 터치 미달.
- **빈 상태 CTA 부재.** 다음 행동으로 못 끈다.
- **결정/일기 회고 분절.** 결정은 Inbox·`decisions`, 일기는 Today·Archive로 갈려 한 시간축 회고가 어렵다(타임라인 세그먼트로 완화 중).

## 3. 핸드메이드 감성 적용 방안 (HOW)

> 원칙(DesignSystem §1-5 준수): **장식은 의미를 거든다.** 가독성·44pt 터치·색각 보조를 침해하지 않는 선에서, "종이에 꽂아둔 기록"이라는 은유를 강화한다. 모든 신규 값은 **토큰에 먼저 추가 후 참조**(하드코딩 금지).

### 3.1 종이 표면에 "질감과 깊이" 주기

평평한 단색 배경이 가장 큰 인상 손실 지점이다. 세 겹으로 표면감을 쌓는다.

1. **캔버스 결 (전역, 1회 작업).** `ScreenBackground`에 미세 노이즈/도트 텍스처를 더한다. 방법: (a) 타일 가능한 반투명 PNG 1장 오버레이, 또는 (b) `react-native-svg`로 저밀도 도트 패턴 + 가장자리 비네팅. opacity 3~6%로 **읽기를 방해하지 않는 결**만. 토큰 `surface.grain`(텍스처 에셋 참조)·`shadow.pageEdge` 신설.
2. **카드 = 떠 있는 종이.** 카드 그림자를 단일 `shadow.card`에서 **이중 그림자**(가까운 contact shadow + 멀고 흐린 ambient)로 바꿔 "들린" 느낌. `paperRaised` 카드엔 미세한 안쪽 하이라이트 보더(상단 1px 밝은 선)로 종이 두께를 암시.
3. **겹침으로 깊이.** 같은 날 여러 클립은 폴라로이드를 **±1~3° 교차 기울임 + 살짝 겹쳐** 쌓아(이미 `PhotoStack` 자산 있음) "사진 더미"를 표현. 맨 위 카드만 선명, 아래는 그림자로.

### 3.2 손글씨를 "주인공 자리"에 정확히 배치

손글씨(Gaegu)는 **작고 의미 있는 곳**에서 빛난다. 대형 헤더 일괄 적용은 역효과.

- **날짜·요일·인사**: 손글씨로. 예 Today 헤더 "토요일"(Gaegu) + 보조 "6월 20일"(산세리프 caption). 큰 산세리프 "Archive" 대신 손글씨 "보관함"으로 톤 통일 검토.
- **섹션 제목에 형광펜.** "검토 대기 · N건", "완료 · N건" 같은 섹션 헤더 텍스트 뒤에 `accent.highlight` 형광펜 띠(텍스트 baseline에 살짝 어긋난 사각형)를 깐다 — 손으로 그은 느낌.
- **본문은 산세리프 유지.** 가독성 영역(클립 본문·라벨·버튼)은 NotoSansKR 그대로. 대비가 핵심.
- **수치·통계 타일**은 손글씨 `displayMedium`로 "직접 적은 숫자" 느낌(Settings 통계·"이번 달 32개").

### 3.3 테이프·압정·스티커를 카드에 실제로 붙이기

`Pin`·`Tape` 컴포넌트가 있으나 거의 미사용. 의미 규칙을 정해 일관 적용한다.

| 장식 | 의미(언제) | 위치 | 토큰 |
|------|-----------|------|------|
| 압정(`Pin`) | "중요/고정된 것" — 중요 결정, On This Day, 고정 메모 | 카드 상단 중앙 | `accent.pin` + `shadow.pin` |
| 마스킹 테이프(`Tape`) | "붙여둔 것" — 폴라로이드 모먼트, 빈 상태 자리표시 | 카드 모서리 1~2곳 | `accent.tape`(반투명) |
| 스티키 태그 | 카테고리(이미 적용) | 카드 좌상단 | `accent.tagBg` |
| 형광펜 | 강조 텍스트 | 텍스트 뒤 | `accent.highlight` |

규칙: **한 카드에 장식 최대 1~2개.** 과하면 산만·터치 방해. 압정/테이프는 카드 콘텐츠 영역 밖(여백)에 얹어 텍스트와 겹치지 않게.

### 3.4 빈 상태 = 가장 큰 감성 기회

빈 화면은 사용자가 가장 오래 보는 화면 중 하나인데 지금은 텍스트 1줄. 종이 메타포로 재설계:

- **손그림 일러스트**(react-native-svg, 단색 잉크 라인): 빈 폴라로이드 프레임 + 마스킹 테이프, 또는 펼친 노트.
- **점선 자리표시**(`border.dashed`) + 압정으로 "여기 꽂으세요" 암시.
- **명확한 CTA**: "오늘의 첫 스냅샷을 남겨보세요" 아래에 **녹화/음성/쓰기로 직행하는 버튼**(캡처바와 연결). 손글씨 마이크로카피로 따뜻하게.
- Archive 빈 달도 동일 패턴 + "다른 달 보기".

### 3.5 마감 디테일

- **구분선**을 직선 hairline 대신 **손그림 느낌**(살짝 굵기 변하는 SVG 라인, 또는 점선)으로 — 타임라인 세퍼레이터부터.
- **아이콘 톤**: 탭바·주요 액션부터 라운드/약간 굵은 세트 또는 손그림 SVG로 시범 교체(`Icon.tsx` 한 곳만 수정). 종이 정서와 정합 A/B.
- **모서리**: 폴라로이드·스티커는 라운드보다 **살짝 불규칙한 직각**이 더 "물건"답다(미세 차이).

## 4. 애니메이션 / 모션 설계 (무엇을 · 어디에 · 어떻게)

> 토대: `react-native-reanimated@4`(설치·미사용)·`gesture-handler`·`@/lib/motion`·`@/lib/haptics`(no-op). **새 라이브러리 없이** 가능. 타이밍은 모션 토큰 사용(`instant 100·fast 150·base 250·slow 400`), 트렌드 합의(마이크로 100–250ms·전환 300–600ms)와 일치.

### 4.0 선행 작업 (모션의 전제)

1. **햅틱 켜기 (P0, 최소 비용·최대 체감).** `npx expo install expo-haptics` → `haptics.ts`의 no-op을 주석으로 준비된 활성 구현으로 교체 → 데브클라이언트 리빌드. 발화 지점이 이미 배선돼 있어 **교체 즉시 앱 전체 촉각 ON.**
2. **모션 표준 정하기.** 등장=스프링(`damping 18, stiffness 180` 류) 또는 `decelerate` 이징, 퇴장=`accelerate`. 신규 토큰 `spring.soft`/`spring.bouncy`를 `tokens.ts`에 추가해 Reanimated `withSpring` 설정을 단일화(매직넘버 금지 규칙 유지).

### 4.1 어디에 (우선순위순)

#### 핵심 순간 — "손맛"을 만드는 4곳 (P0)

| 순간 | 시각 모션 | 햅틱 | 비고 |
|------|----------|------|------|
| **저장 완료** | 카드가 아래에서 스프링으로 떠오르며 리스트에 안착 + 체크 마이크로 펄스 | `success` | "기록이 꽂혔다"는 보상. Asana 유니콘의 절제 버전 |
| **결정 컨펌** | 카드가 살짝 커졌다 압정 박히듯 스냅 + 압정 장식 페이드인 | `success` | Inbox 덱/리스트 공통 |
| **결정 기각** | 카드 좌측으로 슬라이드 아웃 + 약간 회전(`accelerate`) | `warning` | 스와이프 덱과 일관 |
| **녹화 시작/정지** | 녹화 점(`media.recordDot`) 부드러운 펄스(호흡) 시작/정지, 버튼 모핑 | `impact` | "녹음되고 있다" 안심 신호 |

#### 전환·등장 (P1)

- **리스트 stagger 등장.** Today 클립·Inbox 카드·Archive 타임라인이 화면 진입 시 위에서 아래로 40~60ms 간격 순차 페이드+슬라이드(`FadeInDown` 류). 정적 점프 → 흐름.
- **탭 전환 크로스페이드.** expo-router 탭 전환에 짧은 페이드(150ms). 즉시 점프 제거.
- **시트/모달 일관.** 시트=slide-up + 뒤 스크림 페이드, 다이얼로그=fade+scale(0.96→1). `@/lib/motion` 또는 native `animationType`로 통일(이미 규칙 존재 — 적용 점검).
- **세그먼트/필터 전환.** 캘린더↔주↔타임라인, 필터 칩 토글에 `layoutAnimate`(이미 있음) 일관 적용 + `selection` 햅틱.

#### 비동기 상태 (P1)

- **잡 큐 진행 표시.** 압축→STT→라벨추출 파이프라인을 카드에 **부드러운 진행 인디케이터**(맥동하는 점선/형광펜이 차오르는 막대)로. 완료 시 페이드 교체.
- **스켈레톤/플레이스홀더.** 로딩 중 빈 폴라로이드 프레임이 은은하게 shimmer(종이 결 흐르듯). 갑작스런 팝인 제거.
- **STT 텍스트 등장.** 변환된 텍스트가 타이핑되듯 또는 라인별 페이드로 나타나면 "방금 받아적었다"는 감각.

#### 제스처·미세 인터랙션 (P2)

- **녹음 파형/레벨 미터.** `record-audio`에 실시간 진폭 바(Reanimated `useSharedValue`로 60fps). 음성-우선 앱의 기본 신뢰 요소.
- **당겨서 새로고침.** Archive/Inbox에 고무줄→스냅→체크 + 햅틱(트렌드 표준 패턴).
- **버튼 press 물리.** Primary 버튼 press 시 scale 0.96 스프링(현재 색만 변함). 모든 `Pressable`에 공용 `<PressableScale>` 래퍼로 일괄.
- **압정/폴라로이드 롱프레스.** 살짝 들리며 그림자 깊어지는 "집어드는" 피드백 → 정렬/삭제 진입.

### 4.2 어떻게 (구현 패턴)

- **공용 컴포넌트화.** `<PressableScale>`·`<AppearIn>`(stagger 등장)·`<Pulse>`(녹화 점/진행)를 `src/components/ui`에 두고 화면은 의미만 지정 — 토큰·아이콘·햅틱과 같은 "단일 진입점" 패턴 유지.
- **Reanimated 3/4 API.** `withSpring`/`withTiming`(토큰 spring/duration 참조), 진입은 `entering={FadeInDown.springify()}`. 60fps 워크릿으로 JS 스레드 부담 0.
- **햅틱은 의미 어휘로만.** `haptics.success/impact/...` 호출, raw `expo-haptics` 금지(규칙 준수). **과용 금지** — 확정 순간에만.
- **접근성.** OS "동작 줄이기(reduce motion)" 감지 시 등장/스프링을 즉시 페이드로 대체하는 가드를 `motion` 헬퍼에 1곳 추가.
- **검증.** 각 변경 후 `npm run verify` + 에뮬레이터 시각 확인. 모션은 60fps 체감 + reduce-motion 폴백 확인.

## 5. 정보구조 / 사용흐름 개선

> 6/19 흐름 계획 중 **미반영 잔여 항목**과 감성/모션과 맞물리는 구조 조정만.

1. **헤더 타이포 절제 (P0, 토큰 1곳).** 탭 헤더 기본을 `displayLarge`(34)→`displayCompact`(24~26)로. `displayLarge`는 환영 등 1곳 한정. (선택) 스크롤 시 축소되는 sticky compact 헤더.
2. **Today 날짜 위계 단일화 (P0).** 헤더=의미 라벨(오늘/어제/요일, 손글씨) + 보조=전체 날짜 1회. 월·일 중복 제거.
3. **이동/네비 44pt + 시인성 (P0).** Today chevron·Archive 월이동을 면적 있는 pill 버튼(`surface.sunken` + `text.secondary`↑, 44pt)으로. 비활성만 흐리게. 라벨 탭→피커 점프.
4. **빈 상태 CTA (P1).** §3.4와 결합 — 첫 사용자 이탈 방지.
5. **결정·일기 한 축 회고 (P2).** Archive 타임라인에 결정 카드를 같은 시간축에 인레이(일기+결정 통합 회고). `decisions` 분리 화면은 필터로 흡수 검토.
6. **(향후) AI 보조 진입점 자리 잡기.** 결정 컨펌 화면 하단에 "비슷한 과거 결정 N건" 인레이 슬롯만 미리 확보(별도 탭 자제). "추천"이 아닌 **"참고용 과거 패턴"** 마이크로카피로 과신 방지.

### 권장 진행

1. **불부터 켠다 — 햅틱(P0).** 한 줄 설치 + 교체로 앱 전체 촉각 ON. 즉시 체감, 위험 0.
2. **핵심 4순간 모션 + 빈 상태**를 한 스프린트로. "손맛 + 첫인상"을 같이 올린다.
3. **공용 모션 컴포넌트**(`PressableScale`/`AppearIn`/`Pulse`)를 먼저 만들고 화면에 뿌린다(토큰·아이콘과 같은 단일 진입점 규율 유지).
4. **감성 마감**(질감·장식·손글씨)은 토큰·공용 컴포넌트 수정으로 전 화면 동시 반영.
5. 각 항목 `feature-dev` 하네스로 ADR 준수 구현, `npm run verify` 게이트 통과 + 에뮬레이터 확인. 신규 토큰(spring·grain·shadow)은 **토큰 추가 후 참조.**
